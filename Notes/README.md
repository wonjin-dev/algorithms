## DP (Dynamic Programming)

> $O(N^2)$

- `최댓값 / 최솟값 / 경우의 수`를 구할때 자주 사용하는 알고리즘
- 그리디로 풀 수 있는 문제는 DP로도 풀 수 있다.
  - 시간복잡도에서 손해를 봄
- Input이 `5,000이하`인 경우가 많다.
  - 일반적으로 DP가 `O(N^2)`의 시간 복잡도를 가질때, `N = 5,000`이면 2,500만번의 연산을 한다. 이는 자바스크립트 1초 연산의 마지노

```
const dp = new Array(x + 1).fill(Inifinity);
```

## Greedy

> $O(N \log N)$

- `최적해`를 구할때 사용하는 알고리즘
- `정렬`과 `우선순위 큐`가 자주 함께 사용된다.
